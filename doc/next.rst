
Next
====

If you are looking for further reading, the best next place is the
`examples page
<https://github.com/EngineeringSoftware/gobash/tree/main/examples/README.md>`_.

Regarding the features in gobash, there is a lot of potential future
work: improving flag parsing, API extensions, etc.  Regardless what
path this repo takes next, it should always keep programming
abstractions simple (e.g., no hiding anything behind annotations).

Performance is the current biggest issue. We have several ideas on
improving the performance, but we need some available time.
